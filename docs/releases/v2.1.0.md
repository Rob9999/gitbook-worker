# GitBook Worker v2.1.0 Release Notes

**Release Date**: January 12, 2026 (planned)  
**Tag (GitHub)**: release-v.2.1.0  
**Packaging Version**: 2.1.0

## 🎯 Summary
Minor version release adding automated font attribution generation and comprehensive multilingual example content for testing and documentation purposes. The font attribution generator ensures license compliance by automatically creating `ATTRIBUTION.md` and `LICENSE-*` files from the centralized `fonts.yml` configuration.

## ✨ New Features

### Font Attribution Generator
- **Automated License Compliance**: New `generate_attribution` workflow step creates attribution and license files automatically from `fonts.yml`
- **Output Files**: Generates `ATTRIBUTION.md`, `LICENSE-CC-BY-4.0`, and `LICENSE-BITSTREAM-VERA` in publish output directories
- **Offline/Deterministic**: License texts extracted from repository's `LICENSE-FONTS` file without external dependencies
- **CLI Tool**: New `python -m gitbook_worker.tools.publishing.fonts_cli generate-attribution` command for manual generation
- **Configuration**: Controlled via `generate_attribution: true` in `publish.yml` targets
- **Documentation**: Complete usage guide in `docs/feature/font-attribution.md`
- **Commit**: 0963cb1 "Add font attribution generator; bump version to 2.1.0"

### Project Version on PDF Title Page
- **Version Metadata**: New `project.version` field in `publish.yml` is rendered on the PDF title page alongside the publication date
- **Flexible Format**: Accepts semver strings (`1.2.0`), prefixed versions (`v1.0`), or free-form labels (`Draft 1`); numeric YAML scalars (`1.0`) are handled gracefully
- **Dual Source**: Reads version from `publish.yml` (`project.version`) with fallback to `book.json` (`"version"`), same precedence model as `date`
- **Combined Rendering**: Date and version are joined with a middle-dot separator on the title page, e.g. `2026-01-08 · Version 1.0.0`
- **Fully Optional**: Version remains optional; omitting it preserves the existing date-only behaviour

  | Configuration | Title page shows |
  |---|---|
  | `date` + `version` | **2026-01-08 · Version 1.0.0** |
  | only `date` | **2026-01-08** |
  | only `version` | **Version 1.0.0** |
  | neither | *(no date line)* |

## 📌 Migration Notes

### For Existing Users
1. **Project Version on Title Page** (optional):
   - Add `version: "1.0.0"` under the `project:` section of `publish.yml`
   - Alternatively set `"version": "1.0.0"` in `book.json`
   - The version will appear on the PDF title page as `<date> · Version <version>`
   - No change needed if you don't want a version on the title page

2. **Font Attribution** (optional):
   - Add `generate_attribution: true` to publish targets in `publish.yml` to enable automatic license file generation
   - Ensure `fonts.yml` has complete metadata (`license`, `license_url`) for all fonts
   - Attribution files will appear in `<out_dir>/ATTRIBUTION.md` and `<out_dir>/LICENSE-*`

3. **Example Content**:
   - New example files available for reference/testing purposes
   - No breaking changes to existing content structures
   - All examples follow established frontmatter conventions

4. **No Breaking Changes**: This is a minor version bump; all existing workflows remain compatible
