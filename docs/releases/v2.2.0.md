# v2.2.0 „Lückenlos" (8. Februar 2026)

> **Codename**: „Lückenlos" — Dieses Release schließt alle Lücken in
> Konfigurationsdokumentation, Versionierung, Testabdeckung und
> Sample-Content-Strategie.

## Übersicht

v2.2.0 führt eine vollständige **Config-Completeness-Policy** ein, die
sicherstellt, dass jeder Konfigurationsschlüssel im gesamten Projekt einen
dokumentierten Status hat (✅ Implementiert, 🔨 Teilweise, 📝 Deklarativ,
🚧 WIP, ❌ Unused). Dazu gehören Per-File-Konfigurationsdokumentation,
ein umfassender Audit-Backlog und neue Regeln in AGENTS.md.

## Neue Features

### Config-Completeness-Policy (AGENTS.md §25–30)

- **§25**: Jeder Konfigurationsschlüssel braucht einen dokumentierten Status
- **§26**: Neue Schalter nur mit gleichzeitiger Dokumentation
- **§27**: WIP-Schalter müssen warnen, nie still fehlschlagen
- **§28**: Testpflicht für alle ✅-Schalter
- **§29**: Jede Config-Datei muss ein `version`-Feld tragen (SemVer);
  Schema-Änderungen werden in `docs/configs/` dokumentiert
- **§30**: Sample-Content-Abdeckung in `de/` und `en/` für jeden ✅-Schalter;
  Feature-spezifische Sprachbäume für Edge Cases

### Konfigurationsreferenz

- [docs/configuration-reference.md](../configuration-reference.md) —
  Kompaktreferenz aller Schlüssel mit Status-Legende und Versionen-Tabelle

### Per-File-Konfigurationsdokumentation (docs/configs/)

Für jede Konfigurationsdatei existiert ein eigenes Dokument mit:
- Zweck und Ort
- Schema-Version und Versionshistorie
- Vollständige Schlüssel-Tabelle (Typ, Pflicht, Default, Status, Beschreibung)
- Beispiel
- Offene Punkte

| Dokument | Config-Datei | Schema-Version |
|----------|-------------|----------------|
| content-yaml.md | `content.yaml` | 1.0.0 |
| publish-yml.md | `publish.yml` | 0.1.1 |
| book-json.md | `book.json` | – (kein Schema-Feld) |
| fonts-yml.md | `fonts.yml` | 1.0.0 |
| frontmatter-yml.md | `frontmatter.yml` | 1.0.0 |
| readme-yml.md | `readme.yml` | 1.0.0 |
| smart-yml.md | `smart.yml` | 1.0.0 |
| docker-config-yml.md | `docker_config.yml` | – (fehlt!) |
