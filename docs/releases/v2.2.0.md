## 🇩🇪 Deutsch

# v2.2.0 „Lückenlos" (8. Februar 2026)

> **Codename**: „Lückenlos" — Dieses Release schließt alle Lücken in
> Konfigurationsdokumentation, Versionierung, Testabdeckung und
> Sample-Content-Strategie.

## Übersicht

v2.2.0 führt eine vollständige **Config-Completeness-Policy** ein, die
sicherstellt, dass jeder Konfigurationsschlüssel im gesamten Projekt einen
dokumentierten Status hat (✅ Implementiert, 🔨 Teilweise, 📝 Deklarativ,
🚧 WIP, ❌ Unused). Dazu gehören Per-File-Konfigurationsdokumentation,
ein umfassender Audit-Backlog und neue Regeln in AGENTS.md.

## Neue Features

### Config-Completeness-Policy (AGENTS.md §25–30)

- **§25**: Jeder Konfigurationsschlüssel braucht einen dokumentierten Status
- **§26**: Neue Schalter nur mit gleichzeitiger Dokumentation
- **§27**: WIP-Schalter müssen warnen, nie still fehlschlagen
- **§28**: Testpflicht für alle ✅-Schalter
- **§29**: Jede Config-Datei muss ein `version`-Feld tragen (SemVer);
  Schema-Änderungen werden in `docs/configs/` dokumentiert
- **§30**: Sample-Content-Abdeckung in `de/` und `en/` für jeden ✅-Schalter;
  Feature-spezifische Sprachbäume für Edge Cases

### Konfigurationsreferenz

- [docs/configuration-reference.md](../configuration-reference.md) —
  Kompaktreferenz aller Schlüssel mit Status-Legende und Versionen-Tabelle

Alle Standard-Pandoc/LaTeX-Variablen werden nun transparent aus `pdf_options` an
Pandoc weitergereicht. Kunden können damit ihre PDFs vollständig konfigurieren:

### Neue pdf_options-Schlüssel

| Schlüssel | Typ | Beschreibung |
|-----------|-----|--------------|
| `documentclass` | string | LaTeX-Dokumentklasse (`report`, `book`, …) |
| `fontsize` | string | Schriftgröße (`11pt`, `12pt`, …) |
| `geometry` | string | Seitengeometrie (`a4paper,margin=2.5cm`) |
| `toc` | bool | Inhaltsverzeichnis übersteuern |
| `toc-depth` | int | TOC-Tiefe |
| `numbersections` | bool | Abschnittsnummerierung |
| `colorlinks` | bool | Farbige Hyperlinks |
| `linkcolor` | string | Farbe interner Links |
| `urlcolor` | string | Farbe externer URLs |
| `citecolor` | string | Zitationslink-Farbe |
| `toccolor` | string | TOC-Link-Farbe |
| `lang` | string | Sprache (überschreibt `book.json:language`) |
| `header-includes` | string | Raw-LaTeX für Dokumentpräambel |
| `classoption` | string | Zusätzliche LaTeX-Klassenoptionen |
| `papersize` | string | Papiergröße |
| `linestretch` | string | Zeilenabstandsfaktor |
| `mainfont`/`sansfont`/`monofont` | string | Pandoc-native Font-Keys (Alternative zu `main_font` etc.) |

# v2.2.0 “Seamless” (8 February 2026)

> configuration documentation, versioning, test coverage, and

## Overview

v2.2.0 introduces a comprehensive **config completeness policy** that
ensures every configuration key across the entire project has a clearly
documented status (✅ implemented, 🔨 partial, 📝 declarative,
🚧 WIP, ❌ unused). This includes per-file configuration documentation,
a complete audit backlog, and new rules defined in `AGENTS.md`.

## New Features

### Config Completeness Policy (AGENTS.md §§25–30)

* **§25**: Every configuration key must have a documented status
* **§26**: New switches may only be introduced together with documentation
* **§27**: WIP switches must emit warnings and must never fail silently
* **§28**: Tests are mandatory for all ✅ switches
* **§29**: Every config file must contain a `version` field (SemVer);
  schema changes are documented in `docs/configs/`
* **§30**: Sample content coverage in `de/` and `en/` for every ✅ switch;
  feature-specific language trees for edge cases

## pdf_options Passthrough (New in This Release)

All standard Pandoc/LaTeX variables are now transparently passed through
from `pdf_options` to Pandoc. This allows users to fully configure their PDFs:

### New `pdf_options` Keys

| Key                                  | Type   | Description                                                |
| ------------------------------------ | ------ | ---------------------------------------------------------- |
| `documentclass`                      | string | LaTeX document class (`report`, `book`, …)                 |
| `fontsize`                           | string | Font size (`11pt`, `12pt`, …)                              |
| `geometry`                           | string | Page geometry (`a4paper,margin=2.5cm`)                     |
| `toc`                                | bool   | Override table of contents                                 |
| `toc-depth`                          | int    | TOC depth                                                  |
| `numbersections`                     | bool   | Section numbering                                          |
| `colorlinks`                         | bool   | Coloured hyperlinks                                        |
| `linkcolor`                          | string | Internal link colour                                       |
| `urlcolor`                           | string | External URL colour                                        |
| `citecolor`                          | string | Citation link colour                                       |
| `toccolor`                           | string | TOC link colour                                            |
| `lang`                               | string | Language (overrides `book.json:language`)                  |
| `header-includes`                    | string | Raw LaTeX for document preamble                            |
| `classoption`                        | string | Additional LaTeX class options                             |
| `papersize`                          | string | Paper size                                                 |
| `linestretch`                        | string | Line spacing factor                                        |
| `mainfont` / `sansfont` / `monofont` | string | Pandoc-native font keys (alternative to `main_font`, etc.) |

### New Aliases

| Old (user-facing) | New (accepted)      | Description                                      |
| ----------------- | ------------------- | ------------------------------------------------ |
| `project.author`  | → `project.authors` | Singular string converted to array               |
| `format: pdf`     | → `out_format: pdf` | `format` and `target_format` accepted as aliases |
