# GitBook Worker v2.4.1 Release Notes

**Release date:** May 6, 2026  
**Focus:** Docker packaging hotfix for pip/sdist customer delivery.

## Summary

Version 2.4.1 is a narrow hotfix for customer CI pipelines that build the
publishing Docker image from the delivered Worker package. The 2.4.0 sdist did
not include `Dockerfile.dynamic`, the legacy `Dockerfile`, `Dockerfile.python`,
or `gitbook_worker/tools/requirements.txt`. As a result, workflows that derive
their Docker build inputs from the release package could not build the dynamic
image without out-of-band files.

## Fixes

- Include Docker build inputs in the release package:
  - `gitbook_worker/tools/docker/Dockerfile`
  - `gitbook_worker/tools/docker/Dockerfile.dynamic`
  - `gitbook_worker/tools/docker/Dockerfile.python`
  - `gitbook_worker/tools/requirements.txt`
- Fix `run_docker.py build --use-dynamic` to resolve the Dockerfile from the
  package-local `gitbook_worker/tools/docker/` directory instead of the old
  `.github/gitbook_worker/tools/docker/` path.
- Add packaging regression tests that assert the Docker build inputs are kept in
  `MANIFEST.in`.

## Customer Impact

Customers using `v2.4.0` successfully for local non-Docker builds can continue
to do so. Customers whose CI builds Docker images from the release tarball or
installed package should upgrade to `v2.4.1`.

## Expected CI Path

After unpacking the sdist or installing the package, the dynamic Dockerfile is
available under:

```text
gitbook_worker/tools/docker/Dockerfile.dynamic
```

For package-driven CLI use:

```powershell
python -m gitbook_worker.tools.docker.run_docker build --use-dynamic --rebuild
```
