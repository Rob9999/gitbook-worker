# GitBook Worker v2.4.3 Release Notes

**Release date:** May 7, 2026  
**Focus:** Windows font-stub guard and PDF H4/H5 heading hotfix for customer acceptance follow-up.

## Summary

Version 2.4.3 is a focused customer hotfix after the v2.4.2 PDF acceptance
check. It keeps the CJK line-breaking and ERDA script-font routing from v2.4.2,
then hardens the remaining Windows font-stub risk and fixes Pandoc H4/H5 PDF
heading layout without adding new LaTeX package requirements.

## Fixes

- Reject invalid `luaotfload-tool --find` results when the resolved font file is
  too small or does not start with a plausible TTF/OTF/TTC signature.
- Load `ERDA CC-BY Indic` and `ERDA CC-BY Ethiopic` script helpers from
  validated managed font files using explicit `fontspec` `Path=...` file
  references.
- Stop emitting optional `\IfFontExistsTF{ERDA CC-BY Indic}` and
  `\IfFontExistsTF{ERDA CC-BY Ethiopic}` family probes for script helpers, so
  stale 5-byte Windows user-font stubs cannot abort LaTeX header evaluation.
- Keep script helper macros as no-op fallbacks when no valid managed font file
  exists, preserving documents that do not require those scripts.
- Render Pandoc H4/H5 headings (`\paragraph` and `\subparagraph`) as block
  headings via package-free `\@startsection` redefinitions. The final fix avoids
  `titlesec.sty`, which is not guaranteed in all supported TeX environments.
- Refresh DE/EN sample PDF smoke artifacts after the header changes.

## Customer Impact

Customers building PDFs on Windows should upgrade from `v2.4.2` to `v2.4.3` if
local user-font directories may contain stale ERDA Indic/Ethiopic font stubs.
The release is also recommended for books with Markdown `####` or deeper
headings, because those headings now render as standalone PDF block headings
instead of LaTeX run-in headings.
