# GitBook Worker v2.7.0 Release Notes

**Release date:** May 8, 2026  
**Focus:** Wide Markdown table paper selection for PDF builds.

## Summary

Version 2.7.0 improves how GitBook Worker handles broad Markdown pipe tables in
PDF output. The previous strategy primarily used the number of columns as a
proxy for required width. That was too coarse for customer tables with fewer
columns but long German headers and long cell text: a 9-column table could be
placed on a page that was wide enough by column count, yet too narrow for the
actual content.

The Markdown preprocessor now estimates per-column text width, adds table-cell
padding and a safety factor, and compares the result with the usable text width
after margins. The existing column-count heuristic remains as a lower bound, so
older wide-table behavior is preserved where it was already appropriate.

## Added

- Content-width based paper selection for Markdown pipe tables.
- Usable-text-area checks based on paper width minus left/right margins.
- Anonymized German and English wide-table stress samples with 9 columns, long
  headers, and long cell text.
- Focused unit coverage for wide-table text-width selection and oversize
  fallback behavior.

## Changed

- Wide-table page selection now prefers the paper candidate that satisfies the
  larger of the column-count heuristic and the estimated content width.
- Oversized tables that exceed all supported standard paper sizes keep the
  existing fallback strategy and emit a warning instead of pretending that A1
  fully solves the layout.
- Regenerated German and English sample PDFs include the anonymized wide-table
  regression pages.

## Compatibility Notes

- No new customer configuration key is required.
- Existing Markdown sources do not need to be rewritten.
- The change is limited to Markdown pipe-table preprocessing before Pandoc PDF
  generation.

## Known Limitations

- Table text width is still heuristic. It is intentionally conservative for
  long natural-language cells, but exact TeX box measurement is not performed
  before Pandoc/LuaLaTeX.
- Tables wider than the supported standard paper set still require editorial
  intervention, content restructuring, or future scaling/diagnostic tooling.
- Existing multilingual sample PDFs can still report known Missing-character
  warnings in LaTeX logs for non-CJK language samples; the configured Emoji and
  CJK font gates pass.
