# GitBook Worker v2.4.0 Release Notes

**Release date:** May 5, 2026  
**Focus:** PDF regression gates, AI-reference hardening, provider reliability,
and customer-driven batch workflows.

## Summary

Version 2.4.0 is planned as a larger stability and QA release because the next
release window may be several months away. The release pulls forward practical
customer feedback from v2.0.1 usage and turns the AI reference checker from an
LLM helper into an auditable, report-first QA tool.

## Highlights

- PDF font gate for release smoke tests:
  - verifies configured emoji and CJK fonts from `fonts.yml`
  - checks CJK text extraction signal
  - optionally scans current LaTeX logs for Missing-character and `.notdef`
    regressions
- AI-reference provider support:
  - OpenAI-compatible chat completions
  - Gemini/GenAI via `genai` and `google-genai`
  - Mistral via `mistral` and `mistral-ai`
- AI-reference reliability:
  - report-only by default; Markdown writes require `--apply`
  - secret redaction for reports and provider error strings
  - static throttling and adaptive `Retry-After` backoff
  - customer-compatible aliases for existing wrappers
- AI-reference batch operation:
  - `--files-list`
  - `--max-tasks`
  - `--resume-from-report`
  - `--max-consecutive-429`
- AI-reference evidence handling:
  - `--as-of-date YYYY-MM-DD`
  - prompt rules against invented access dates, publication dates, publishers,
    titles, or authors
  - inline URL/DOI extraction
  - optional Markdown-link and frontmatter-DOI extraction
  - no-network prechecks for URL, DOI, arXiv, ISBN, and internal Markdown links
  - report statuses `suggested`, `validated`, `failed`, and `rate_limited`
- Docker publishing image:
  - `Dockerfile.dynamic` no longer hardcodes the TeX Live year
  - the image resolves the CTAN-installed TeX Live directory and exposes it via
    `/usr/local/texlive/current`
  - legacy `Dockerfile` remains deprecated and outside the v2.4.0 release path
  - `Dockerfile.python` remains a lightweight Python-test container without
    Pandoc or TeX Live
- Release package hygiene:
  - release package discovery excludes `gitbook_worker.tests`
  - source manifest prunes test fixtures and cache artifacts
  - clean wheel inspection found no bundled tests or `__pycache__` entries

## Customer Upgrade Notes

Existing customer wrappers around `ai_references` can migrate gradually:

- `--delay-seconds` maps to the built-in minimum request interval.
- `--jitter-seconds` maps to throttle jitter.
- `--cooldown-on-429-seconds` maps to fixed fallback backoff when a provider
  does not send `Retry-After`.
- `--max-consecutive-429` limits repeated rate-limit failures and enables a
  clean partial report when `--json-report` is used.
- `AI_API_KEY`, `AI_URL`, and `AI_PROVIDER` are accepted alongside the
  `AI_REFERENCE_*` variables.

- English PDF build and font gate: passed on 2026-05-05; Twemoji and ERDA-CJK
  embedded; positive CJK text extraction signal found.
- Clean package build: `gitbook_worker-2.4.0.tar.gz` and
  `gitbook_worker-2.4.0-py3-none-any.whl` built successfully.
- Wheel smoke test: installed `gitbook_worker-2.4.0-py3-none-any.whl` in a fresh
  temporary virtual environment; imported version `2.4.0`; orchestrator help
  started successfully.
- Privacy sanitizer checks: AI-reference reports redact API keys and token-like
  fields; provider error strings redact raw API keys and `?key=` / `api_key=`
  query values. This is technical secret protection, not a legal EU/GDPR audit.
