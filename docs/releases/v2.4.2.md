# GitBook Worker v2.4.2 Release Notes

**Release date:** May 6, 2026  
**Focus:** PDF font and line-breaking hotfix for customer-visible CJK overflow.

## Summary

Version 2.4.2 is a customer hotfix for PDF typography after visual inspection
found that long CJK text could run into the page margin. The release keeps the
v2.4.1 Docker packaging fixes and adds safer CJK line breaking, explicit ERDA
script-font routing, and clearer DE/EN visual-inspection samples for ERDA CJK,
Indic, and Ethiopic coverage.

## Fixes

- Add the default `cjk-linebreak.lua` Pandoc Lua filter, which inserts safe
  LaTeX breakpoints after CJK characters so long Traditional Chinese, Japanese,
  and Korean passages can wrap naturally in LuaLaTeX PDFs.
- Avoid the previous LuaTeX-ja/JFont route that could crash around the table of
  contents in this project setup.
- Keep the global fallback stack stable with CJK before Indic/Ethiopic, avoiding
  the LuaHBTeX HarfBuzz nil-character crash observed when Indic/Ethiopic were
  moved ahead of CJK globally.
- Add `erda-script-fonts.lua` and publisher-side LaTeX helpers so Devanagari and
  Ethiopic samples are routed through `ERDA CC-BY Indic` and
  `ERDA CC-BY Ethiopic` without disturbing the stable global CJK fallback.
- Add anonymized long visual-inspection sections in both DE and EN sample books:
  - `ERDA CC-BY CJK`: Traditional Chinese, Japanese, Korean.
  - `ERDA CC-BY Indic`: Hindi / Devanagari.
  - `ERDA CC-BY Ethiopic`: Amharic and Tigrinya.
- Label each long sample line with flag, language code, and language name for
  faster manual PDF review.

## Customer Impact

Customers building PDFs with long Traditional Chinese, Japanese, Korean,
Devanagari, Amharic, or Tigrinya passages should upgrade from `v2.4.1` to
`v2.4.2`. The release is especially relevant when PDF visual inspection shows
CJK margin overflow or when customer QA needs clear proof that the configured
ERDA fonts are embedded and actively used.
