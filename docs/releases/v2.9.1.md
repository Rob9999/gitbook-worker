# GitBook Worker v2.9.1 Release Notes

**Release date:** May 11, 2026
**Focus:** customer acceptance hardening, URL/DOI PDF wrapping, and more useful editorial quality signals.

## Summary

Version 2.9.1 is a focused hotfix release after the v2.9.0 quality cycle. It
keeps the v2.9.0 quality dossier foundation, then hardens the parts that became
visible during the current German customer acceptance run:

- external or ignored customer roots now run through package-module subprocesses;
- long visible URLs and DOI references are rendered with breakable LaTeX URL
  macros;
- PDF text-layer replacement signals are reported as accessibility/copy-paste
  warnings instead of hard font failures;
- Markdown quality findings are less noisy for editors.

No customer raw content, PDFs, extracted text, or diagnostic dumps are included
in the release artifacts. Customer-derived regressions use anonymized samples.

## Fixed

- Fixed converter preflight execution for ignored/external customer roots by
  invoking `gitbook_worker.tools.publishing.dump_publish` as a package module.
- Added the package source parent to orchestrator subprocess `PYTHONPATH`, so
  module imports stay stable outside the repository root.
- Fixed long visible URL and DOI wrapping in PDFs through `url-breaks.lua` and
  LaTeX URL break configuration.
- Prevented broken nested LaTeX output such as `\href{...}{\url{...}}` for
  visible Markdown URL links.
- Kept access-date markers such as `(Zugriff: ...)` outside generated `\url{...}`
  macros when source Markdown misses a separating space.
- Reclassified PDF text extraction replacement signals as
  `pdf.text.extraction_replacement` warnings.
- Reduced editorial warning noise by ignoring frontmatter and URL-bearing tokens
  in long-token checks.
- Scoped duplicate-heading warnings to nearby repeated headings in the same
  document.
- Required explicit review markers such as `REVIEW:`, `[REVIEW]`, `TODO`,
  `FIXME`, or `XXX`; normal prose such as "peer review" is no longer flagged.

## Added

- Anonymized text-layer replacement fixture under
  `gitbook_worker/tests/data/anonymized/`.
- Regression tests for visible URL links, no-space access markers, external
  customer roots, focused duplicate-heading checks, and explicit review markers.
- FAQ and configuration documentation for URL wrapping and text-layer replacement
  warning semantics.
- Customer-evidence policy in `AGENTS.md`: customer materials are diagnosis-only;
  every customer-derived problem needs an anonymized in-repo regression.

## Known Limitations

- Customer acceptance still reports warnings that require editorial decision or
  content follow-up; v2.9.1 makes those warnings much more focused but does not
  rewrite customer content.
- `pdf.text.extraction_replacement` warns about copy/paste/accessibility text
  extraction quality. It is not a hard visual glyph failure unless a visual PDF
  review or LaTeX log shows missing characters.
- Full customer-side CJK/Hangul/script font expansion remains deferred to the
  v2.10.0 ERDA font program.
- No Noto fonts are used or recommended. Apart from DejaVu, project fonts and
  emoji expansion remain on the CC BY 4.0 ERDA/Twemoji path.
