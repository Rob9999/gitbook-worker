# GitBook Worker v2.9.0 Release Notes

**Release date:** May 10, 2026
**Focus:** honest release evidence, customer-safe delivery notes, and robust PDF checklist symbols.

## Summary

Version 2.9.0 closes the quality hardening cycle with auditable release evidence
and a focused PDF font fix for checklist and checkmark text symbols. The release
keeps the existing Twemoji Mozilla path for color emoji, but treats `☐`, `☑`,
`☒`, `✓`, and `✔` as text symbols so they are no longer pushed into the emoji
font path or Pandoc's math-list fallback.

The recommended fallback chain now keeps `DejaVu Sans:mode=harf` between
Twemoji Mozilla and ERDA script/CJK fallback fonts. This is documented in the
FAQ, README, configuration reference, customer guide, and the German/English
sample content.

## Added

- `text-symbols.lua` Pandoc filter for LaTeX output. It routes checklist and
  checkmark text symbols through the configured sans font via `\erdaTextSymbol`.
- Publisher font header macro `\erdaTextSymbol`, backed by `\sffamily`.
- Regression coverage proving text checkboxes are not wrapped as emoji and do
  not become math labels such as `\square` or `\boxtimes`.
- German and English sample task-list entries that exercise the text-symbol
  fallback path.
- FAQ and delivery documentation for `DejaVu Sans:mode=harf` in project-level
  fallback overrides.

## Changed

- The default Lua filter stack now runs `text-symbols.lua` after emoji span
  detection and before LaTeX emoji substitution.
- `emoji-span.lua` excludes checklist/checkmark text symbols from broad Unicode
  emoji-like ranges.
- German and English `publish.yml` fallback stacks include
  `DejaVu Sans:mode=harf` before `ERDA CC-BY CJK:mode=harf`.
- Own DE/EN sample publish artifacts were regenerated after the sample updates.

## Compatibility Notes

- Existing projects without custom fallback overrides inherit the fixed stack
  from the shipped sample/default configuration path.
- Projects that override `mainfont_fallback` must keep
  `DejaVu Sans:mode=harf` in the chain when task lists or checkbox/checkmark
  symbols are used.
- Twemoji Mozilla remains the color emoji font. A custom ERDA Emoji font is not
  part of v2.9.0 because current Twemoji Mozilla coverage is sufficient for the
  verified emoji corpus.

## Known Limitations

- Full customer-side CJK/Hangul/script font expansion is intentionally deferred
  to the v2.10.0 ERDA font program.
- ERDA Emoji remains a future design option, not a v2.9.0 dependency.
- The local `.venv` used for release checks did not contain `isort` or
  `flake8`; `black --check`, focused regressions, the full non-slow suite,
  package build, wheel smoke, and DE/EN PDF smokes were used as release gates.
