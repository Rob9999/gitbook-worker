# GitBook Worker v2.5.0 Release Notes

**Release date:** May 7, 2026  
**Focus:** ERDA fallback-font coverage, coverage matrix documentation, and TTF-level glyph statistics.

## Summary

Version 2.5.0 turns the remaining ERDA font-coverage backlog into measurable
release targets. It separates long sample text coverage from real TTF glyph
coverage, expands the generated fallback fonts, and adds statistics gates so
future release checks report actual `maxp.numGlyphs`, `cmap` mappings,
per-script Unicode range counts and the complete long-text inspection section.

## Added

- New generator coverage targets for `erda-ccby-cjk.ttf`:
  - at least 3000 CJK Unified Ideographs,
  - at least 3000 Hangul syllables,
  - complete assigned Hiragana and Katakana block coverage,
  - complete ZH-Hant/JA/KO 3000-character sample block coverage,
  - complete CJK-family coverage for the full long-text inspection section.
- Full assigned Devanagari main/extended block targets for
  `erda-ccby-indic.ttf`.
- Full assigned Ethiopic main, supplement, extended, extended-A and extended-B
  block targets for `erda-ccby-ethiopic.ttf`.
- `font_cli.py stats`, backed by `fontTools.ttLib.TTFont`, to inspect generated
  TTF files and fail if known ERDA fonts miss their release targets.
- Deterministic ERDA-owned codepoint-marker glyphs for fallback coverage when a
  handcrafted bitmap is not available.
- TTF-level regression tests for the ERDA font release targets.
- Independent ERDA generated font-family versioning, now at `1.4.1` for CJK,
  Indic and Ethiopic TTFs.
- Coverage matrix documentation in `.github/fonts/erda-ccby-cjk/docs/`.

## Changed

- Generated ERDA marker glyphs now use sparse, licensed marker shapes instead
  of missing-glyph-style black boxes.
- Generated marker ink was raised to `0.90` cell width for better PDF
  readability in the 3000-character inspection blocks.
- Mixed Devanagari/Ethiopic text runs are split before applying explicit ERDA
  script font macros, so surrounding Latin text and punctuation remain in the
  main text font instead of producing `.notdef` boxes.

## Quality Boundary

The 2.5.0 expansion is a licensed fallback-font hardening step. It prevents
missing glyph boxes and makes coverage measurable, but it does not claim to be a
full-quality CJK, Indic or Ethiopic typeface. Handcrafted bitmap glyphs remain
preferred, and future full text-quality work still requires type design,
metrics, hinting, kerning, shaping validation and wider layout regression gates.
