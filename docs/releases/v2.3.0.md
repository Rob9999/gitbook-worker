## ЁЯЗмЁЯЗз English

### Summary

Version 2.3.0 "Pfadtreu" (Path-True) fixes the critical customer bug where the
GitBook rename step (`gitbook_style.py rename`) destroyed file paths when
publishing individual files (`source_type: file`).

### Changes

#### ЁЯЫбя╕П GitBook Rename Auto-Detect & Explicit Key

- **Auto-Detect**: When **all** publish entries in the manifest have
  `source_type: file`, the rename step is automatically skipped. This
  preserves directory and file names exactly as declared in `publish.yml`.
- **New top-level key `gitbook_rename`**: Can be set to `false` to explicitly
  disable the rename step тАФ even with mixed entries. Example:

  ```yaml
  gitbook_rename: false
  ```

- **Priority**: CLI flag `--no-gitbook-rename` > manifest key `gitbook_rename` > auto-detect.

#### ЁЯФз `--no-gitbook-rename` on Orchestrator

- The CLI flag `--no-gitbook-rename` is now available on the **orchestrator**
  and forwarded to `pipeline.py`.

#### ЁЯРЫ Fix: RawFeature fallback despite `ERDA_ENABLE_LUA_FALLBACK=0`

- **Bug**: `\setmainfont`, `\setsansfont` and `\setmonofont` received
  `[RawFeature={fallback=mainfont}]` even when `ERDA_ENABLE_LUA_FALLBACK=0`
  was set. Since the Lua fallback table was never registered, LuaLaTeX
  crashed with rc=1.
- **Fix**: Both locations in `_build_font_header()` now check
  `enable_lua_fallback` before emitting `RawFeature`.
- **4 regression tests** in `test_font_fallback_rawfeature.py`.

#### ЁЯУж Manifest Version 0.1.1

- `_MANIFEST_VERSION_CURRENT` in the publisher updated from `(0, 1, 0)` to
  `(0, 1, 1)`. Customers whose `publish.yml` already contains `version: 0.1.1`
  will no longer see the "newer than tested" warning.

#### ЁЯУЦ FAQ Document

- New [docs/FAQs.md](../FAQs.md) with 5 frequently asked questions from
  real customer debugging sessions:
  1. FileNotFoundError after GitBook rename
  2. Missing `project.license`
  3. Profile fallback to `default`
  4. Manifest version warning
  5. Special characters in filenames

#### ЁЯзк Tests

- 7 new unit tests for `_should_skip_rename()` in `test_pipeline.py`:
  - Explicit `gitbook_rename: false/true`
  - Auto-detect with all `source_type: file`
  - Mixed entries тЖТ no skip
  - Empty publish list тЖТ no skip
  - Explicit key overrides auto-detect
- 4 regression tests for RawFeature fallback in `test_font_fallback_rawfeature.py`.
- **Test status**: 463 passed, 11 skipped, 0 failures.
