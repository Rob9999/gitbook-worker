# GitBook Worker v2.6.1 Hotfix Release Notes

**Release date:** May 8, 2026  
**Focus:** URL wrapping inside PDF code fences.

## Summary

Version 2.6.1 is a focused hotfix for a customer PDF layout case found after
v2.6.0. Long URL-like tokens inside fenced code blocks were still able to run
past the right page boundary because Pandoc rendered them as highlighted token
macros such as `\NormalTok{...}`. The v2.6.0 `fvextra` environment wrapping was
active, but the macro argument itself remained too rigid for reliable breaking.

The publisher now routes Pandoc token macro arguments through `fvextra` break
insertion when the package supports it. This keeps long URLs inside code fences
within the page bounds without rewriting source Markdown and without adding
visible wrap symbols to copied PDF text.

## Fixed

- Long URLs inside Pandoc-highlighted code fences now wrap across lines instead
  of clipping or overflowing beyond the PDF text block.
- The generated LaTeX header clears `fvextra` break-anywhere symbols for this
  path so PDF text extraction remains clean.
- German and English sample PDFs were regenerated with URL stress cases that
  exercise the fixed behavior.

## Compatibility Notes

- Existing manifests keep the behavior controlled by
  `pdf_options.code_block_wrap`, which remains enabled by default.
- The hotfix still degrades safely when `fvextra.sty` is unavailable: builds
  continue with code wrapping skipped and a package warning.
- The fix does not rewrite customer Markdown content. It only changes the
  generated LaTeX header used for PDF builds.

## Known Limitations

- This hotfix targets code-fence URL tokens. Wide Markdown tables are tracked
  separately because they require reliable measurement and paper/strategy
  selection rather than verbatim token wrapping.
